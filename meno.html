Strategy 01
模版分发体系优化
通过场景化导航重构 + 行为信号自动排序算法，解决生图类内容曝光不足问题，上线后首页各模块点击率与功能人均使用频次显著提升
📈
核心成果
首页总点击率 ↑ ~22pp  ·  生图场景曝光占比 翻升  ·  功能人均使用频次 同步提升
推荐算法内容分发优化场景化标签设计行为信号加权排序
痛点洞察
首页内容流量分布与使用频次呈负相关，其中生图类场景曝光显著不足，导致用户进入首页后未触发制作行为即流失。当前分发机制是按功能类型而非使用场景组织。
策略推导逻辑
数据观察
各场景流量分布与使用频次负相关——生图类曝光占比高，但用户实际使用少
→
归因假设
分发按功能类型组织，与用户真实使用场景不匹配
→
策略选择
场景化重构分发，以行为信号代替人工排序，可量化可迭代
策略设计
1
场景化重构导航
内容按用户使用意图拆为三类：Video / Photo / UGC，对应下发不同素材类型；一级导航升级为 Discover → Templates → AI tools → Rewards
2
行为信号排序算法
整体得分 = λ × 热门得分 热门得分 = （试用渗透率 × W1）+ （保存渗透率 × （1-W1）） λ = 时间衰减因子（控制内容新鲜度），W1初始值0.6，可基于情况调参
3
新内容冷启动保护
阶段机制
上线 24h 内不参与正式排序，按上传时间倒序
第 2–3 天给予 20% 曝光加权，保障最低曝光基础
每日 0 点自动重算，首位优先级 10000，每位递减 10
4
ERP 后台标签管理系统
打通素材上新 → 标签关联 → 排序配置全流程；运营可在后台配置子标签优先级，支持新老用户差异化内容投放，无需前端发版即可调整分发策略
落地分工
前端重构首页导航结构，按场景意图分类下发内容；接入行为信号排序算法，设定自动重算周期
运营在 ERP 后台建立素材标签体系，配置子标签优先级与新老用户差异化投放策略；持续迭代参数权重
数据监控各场景点击率、试用/保存渗透率趋势，作为算法参数调优依据
增长链路
分发效率↑→ 首页找到感兴趣内容→ 点击进入制作→ 完成核心使用动作→ Day1 留存↑

    • Strategy 02
    Prompt输入结构化分层设计
    通过将自由文本输入结构化为多维表单 + 分层Hints，降低新用户制作门槛，结果生成率显著提升，生图功能人均积分消耗 +37.8%
    ⚡
    核心成果
    生图功能人均积分消耗 +37.8%  ·  制作完成率 ↑显著改善  ·  新用户 Aha Moment 完成比例提升
    Prompt结构化设计用户分层引导渐进式信息披露
    痛点洞察
    用户关键行为转化率流失较大，制作点击率与结果生成率为新用户转化路径中的两大主要断点。
    推导逻辑
    数据观察
    "制作点击率"与"生成率"双低——连续两级漏斗断点，流失集中在输入环节
    →
    归因假设
    用户知道想要什么，但无法清晰表达——障碍在输入侧，不在生成侧
    →
    策略选择
    结构化表达而非削减功能，保留深度的同时分层降低门槛
    策略设计
    1
    Prompt 结构化设计
    将开放式文本输入拆解为维度化结构，降低用户表达成本：
    主体 + 动作 （必填） 一个女孩戴着钻石皇冠优雅地旋转 场景 （选填） 一间有大窗户的明亮芭蕾舞教室 光线 / 情绪 （进阶选填） 温暖逆光，梦幻感（可扩展参数）
    2
    写真 AI Agent 功能化
    通过样片结构化解析，输出含风格、姿势、服饰、布景维度的可编辑 Prompt，结合用户照片生成定制化写真。将复杂多步操作封装为 Agent，大幅降低专业用户门槛。
    →上传样片 → 结构化解析（风格 / 姿势 / 服饰 / 布景）
    →输出可编辑 Prompt，用户可微调各维度
    →结合用户照片 → 一键生成定制化写真
    3
    分层 Hints 运营策略
    三种入口是同一生成流程的不同深度，而非三套独立产品：
    用户类型方案路径
    小白预设 Hints → 一键生成选预设 → 自动生成 → 预览导出
    中级Prompt 维度化（主体 / 场景 / 光线）选预设 → 编辑结构化参数 → 迭代
    高阶AI Agent：样片解析 → Prompt 可编辑上传样片 → 解析维度 → 自定义生成
    落地分工
    产品/前端将输入界面改为结构化维度，设计小白/中级/高阶三层入口
    运营建立预设Hints内容库，根据生成满意率持续迭代；观察各维度填写率识别优化信号
    增长链路
    输入门槛↓→ 新用户首次生成成功→ 完成"Aha Moment"→ Day1留存↑

    • Strategy 03
    商业化卡点测试付费墙位置 + 免费试用精准化
    通过付费墙前/后置A/B 测试 + 精准免费试用策略，定位最优转化时机，T3国家付费转化率 +13.8%，ARPU +23%
    💰
    核心成果
    T3 国家（泰国、越南等）付费转化率 +13.8%  ·  ARPU +23%  ·  后置方案胜出
    商业化A/B测试免费试用设计付费转化机制
    痛点洞察
    功能及模板的整体付费转化率偏低，新用户首购贡献不足。
    推导逻辑
    数据观察
    付费成功率偏低，但仅凭数据无法判断根因在哪一环节
    →
    归因假设
    两个独立假设：① 付费时机不对　② 免费试用资源分配低效
    →
    策略选择
    两个假设分别测试，不混变量；三维联动评估，以 LTV 为最终判据
    策略设计
    1
    付费墙前置 vs 后置A/B 测试
    前置使用前出现付费墙
    短路径，高意图用户直接转化
    ⚠ 未建立价值感知，新用户流失率高
    后置体验后触发付费引导
    先建立价值认知，转化更自然
    ⚠ 可能损失高意图用户的即时转化
    胜出规则：付费率 + ARPPU + 次日留存三维同向改善才算方案赢；出现分裂信号时以 LTV 估算为最终判据
    2
    免费试用分层策略
    ✕传统做法：所有新用户默认获得免费试用，成本高，稀释付费转化
    ✓ Predictive Free trial Mechanism
    →监测流失信号（使用频次下降 / 关键行为中断）
    →流失节点前推送免费试用弹窗
    →重激活付费意愿
    →延长生命周期，提升LTV
    落地分工
    ②
    前端实现：前置 = 点击功能时弹窗；后置 = 结果页触发引导；免费试用 = 设定流失触发条件
    ③
    运营配置：弹窗文案突出产品价值；到期前推送转化提醒
    ④
    数据跟踪：三维指标同时监控；额外追踪'免费试用'触达后复访率和付费激活率
    增长链路
    后置 价值感知后付费→ D1 付费转化↑
    免费试用 精准识别流失信号→ 流失前重激活→ LTV↑
